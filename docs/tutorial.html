1사전 준비 — Node.js, Git 설치 확인

CMD(명령 프롬프트)를 열고 다음 명령어를 입력해 설치 여부를 확인합니다.

1-1. Node.js 확인

node --version
npm --version

버전이 표시되면 설치 완료 (예: v20.10.0).

없으면 → https://nodejs.org/ko 에서 LTS 버전 다운로드.

1-2. Git 확인

git --version

버전 나오면 OK (예: git version 2.49.0).

없으면 → https://git-scm.com/download/win

1-3. 코드 에디터 (VS Code 권장)

VS Code 다운로드

💡 팁 Git 설치 시 "Git Bash" 도 함께 설치됩니다. CMD가 어려우면 Git Bash 써도 OK.

2GitHub 계정 + 레포 만들기

2-1. GitHub 가입 (이미 있으면 건너뛰기)

  1. https://github.com/signup 접속
  2. 이메일 → 비밀번호 → 사용자명 입력
  3. 이메일 인증 완료

2-2. 새 레포지토리 만들기

  1. 로그인 후 우측 상단 + → New repository
  2. 레포 이름: 본인 봇 이름 (예: my-team-03-bot)
  3. Public 선택 (Vercel 무료 배포 가능)
  4. "Add a README file" 체크 안 함 (푸시 충돌 방지)
  5. Create repository 클릭

2-3. Git에 GitHub 계정 등록 (최초 1회)

git config --global user.name "본인_GitHub_사용자명"
git config --global user.email "본인_이메일@example.com"

3템플릿 클론 + 로컬 실행

⭐ 박교수님 자습서와 다름 cha-bot-starterkit 을 클론합니다 (Team Edition: 서버 사이드 RAG + Gemma4 무료 백엔드).

3-1. 작업 폴더로 이동

cd C:\projects
mkdir my-bot
cd my-bot

3-2. 템플릿 클론

git clone https://github.com/sungbongju/cha-bot-starterkit.git .
⚠️ 주의 마지막에 . (점) 있음. 현재 폴더에 클론하라는 뜻.

3-3. 기존 .git 폴더 삭제 (내 레포로 새로 시작)

rmdir /s /q .git
git init -b main

3-4. 의존성 설치

npm install

약 1~3분 걸립니다.

3-5. 로컬 실행

npm run dev

브라우저에서 http://localhost:5173 접속 → 봇이 동작하면 성공!

💡 종료 CMD에서 Ctrl + C → Y 입력

4환경 변수 설정 (.env)

⭐ 박교수님 자습서와 다름 OpenAI API 키 불필요. 대신 TEAM_ID(본인 팀 번호)와 VITE_KAKAO_JS_KEY 만 필요.

4-1. .env 파일 생성

VS Code에서 프로젝트 폴더 열기 → 새 파일 .env 생성 → 다음 입력:

# 본인 팀 번호 (01 ~ 16, 박교수님께 받음)
TEAM_ID=03

# 카카오 JavaScript SDK 키 (Step 7에서 설정)
VITE_KAKAO_JS_KEY=your_kakao_js_key_here

6Vercel 배포

⭐ 박교수님 자습서와 다름 OpenAI 키 대신 TEAM_ID 환경변수 설정. (Gemma4는 미들턴 공유 = 키 불필요)

6-1. Vercel 가입

  1. https://vercel.com/signup 접속
  2. "Continue with GitHub" 선택
  3. 권한 승인

6-2. 프로젝트 배포

  1. 대시보드에서 Add New → Project
  2. 본인 GitHub 레포 (my-team-03-bot) 선택 → Import
  3. Framework Preset: Vite 자동 인식
  4. Environment Variables 펼치기 → 다음 2개 추가:
    NameValue
    TEAM_ID03 (본인 팀 번호 2자리)
    VITE_KAKAO_JS_KEYStep 7에서 발급할 키 (지금은 임시값 가능)
  5. Deploy 클릭

6-3. 완료!

1~3분 후 https://my-team-03-bot.vercel.app 같은 URL이 생성됩니다.

💡 자동 재배포 앞으로 GitHub에 푸시할 때마다 Vercel이 자동 재배포합니다.

7카카오 SDK 설정 (로그인 + 공유)

한 번의 셋업으로 두 가지가 활성화됩니다:

  • 카카오 로그인 — 사용자가 봇 첫 방문 시 자동으로 로그인 모달 (AuthModal) 이 뜸. 카카오 계정 또는 이메일로 가입/로그인 가능.
  • 카카오 공유 — 봇을 카카오톡으로 친구에게 공유 가능.
💡 사용자 흐름 학생 봇 접속자 → 자동 로그인 모달 표시 → 카카오 또는 이메일 선택 → 가입 완료 → 봇과 대화.
로그인 기록은 학교 서버에 저장 (채팅 히스토리 추적용).

7-1. Kakao Developers 가입 및 앱 생성

  1. https://developers.kakao.com/ 접속
  2. 카카오 계정으로 로그인
  3. 내 애플리케이션 → 애플리케이션 추가하기
  4. 앱 이름 입력 (예: "분개해 봇") + 사업자명 (본인 이름) → 생성

7-2. JavaScript 키 복사

앱 선택 → 요약 정보 → JavaScript 키 복사 (32자리 영숫자)

7-3. 사이트 도메인 등록 (필수)

  1. 좌측 플랫폼 → Web 플랫폼 등록
  2. 사이트 도메인에 본인 Vercel URL 입력: 
    https://my-team-03-bot.vercel.app
  3. 저장 클릭 — 등록 안 하면 카카오 SDK 작동 안 함

7-4. 카카오 로그인 활성화 (필수)

  1. 좌측 제품 설정 → 카카오 로그인
  2. 활성화 설정: ON
  3. Redirect URI 등록: https://my-team-03-bot.vercel.app/oauth (사용 안 해도 등록 필요)
  4. 동의 항목 설정: 닉네임 필수, 이메일 선택 (사용자가 거부 가능)
⚠️ 학생 흔한 실수 Step 7-4 의 "카카오 로그인 활성화" 안 하면 봇 첫 방문 시 로그인 모달에서 카카오 버튼 누르면 에러 ("앱 정보가 정확하지 않음"). 반드시 활성화하세요.

7-4. Vercel 환경변수 업데이트

Vercel 대시보드 → 본인 프로젝트 → Settings → Environment Variables:

  • VITE_KAKAO_JS_KEY 값을 복사한 JavaScript 키로 교체
  • Save 후 Deployments → 최근 배포 Redeploy
💡 코드 수정 불필요 Team Edition 은 카카오 키를 index.html에 박지 않고 환경변수로 받습니다. Vercel env 만 바꾸면 즉시 반영.

8RAG 데이터 수정 — 나만의 지식 넣기 ⭐

⭐ 박교수님 자습서와 가장 큰 차이 SSH/Node 스크립트 전혀 사용 안 함. 웹 페이지에서 청크 붙여넣고 버튼 클릭만.

8-1. 본인 팀 RAG 관리 페이지 접속

브라우저에서 (팀 번호 03이면):

https://middleton.p-e.kr/finbot/team/03/rag

본인 팀 번호로 바꿔서 접속하세요.

8-2. 청크 형식 이해

JSONL 형식 (한 줄당 하나의 JSON). 예시 (분개해 봇):

{"id":"acc-001","question":"분개가 뭐예요?","answer":"거래를 차변과 대변으로 나누어 기록하는 회계 절차입니다."}
{"id":"acc-002","question":"차변과 대변의 차이?","answer":"차변(왼쪽)은 자산 증가/부채 감소, 대변(오른쪽)은 자산 감소/부채 증가를 기록합니다."}

8-3. 본인 봇용 청크 만들기

  1. 본인 봇 주제 청크 50~100개 작성 (메모장 또는 VS Code)
  2. JSONL 형식: 한 줄에 하나의 JSON
  3. 각 필드:
    • id: 고유 식별자 (예: "acc-001")
    • question: 사용자가 물어볼 만한 자연어 질문
    • answer: 짧고 구체적 (2~5줄)
    • section (선택): 카테고리
💡 AI로 청크 만들기 ChatGPT/Claude에 다음 프롬프트:
"분개해 봇 (회계 학습 친구) 에 쓸 RAG 청크 50개를 JSONL 형식으로 만들어줘. 형식: {"id":"...","question":"...","answer":"..."}"

8-4. 웹 페이지에서 청크 추가

  1. 웹 페이지 (위 URL) 열기
  2. 본인 봇 정보 확인 (팀 번호, 봇 이름, 학생 이름)
  3. 큰 텍스트 영역에 JSONL 청크 붙여넣기
  4. 팀 키 입력 (박교수님께 별도 받음, 예: team-03-9478c414d6f8eab9)
  5. 청크 추가 버튼 클릭
  6. 임베딩 자동 생성 (청크 수 × 0.5초)
  7. "N개 추가됨. 총 N개." 메시지 → 성공!

8-5. 봇에서 확인

본인 봇 사이트 (https://my-team-03-bot.vercel.app) 새로고침 → 질문 → RAG 활용 답변 확인.

⚠️ JSON 형식 오류 주의 한 줄에 하나의 JSON, 큰따옴표 (작은따옴표 X), 마지막 쉼표 없음. 오류 있으면 웹 페이지가 "JSON 신택스 에러" 보여줍니다.
📌 다른 팀에 영향 없음 본인 팀 키로만 본인 팀 파일에 추가됩니다. 16팀 데이터는 서로 완전히 분리되어 있습니다.

9VRoid 아바타 만들기

봇의 3D 캐릭터를 본인 봇 페르소나에 맞게 만듭니다.

9-1. VRoid Studio 다운로드

  1. https://vroid.com/en/studio 접속
  2. Windows / Mac 버전 다운로드 (무료)
  3. 설치 후 실행

9-2. 캐릭터 만들기

  1. New → 기본 캐릭터 선택 (Female/Male Sample)
  2. 좌측 메뉴에서 외형 커스터마이즈 (Face / Hair / Body / Outfit / Accessories)
  3. 본인 봇 페르소나에 맞게 디자인
    • 분개해 → 학생 같은 단정한 캐릭터
    • 상도 → 한복 + 정장 (시대 혼합)
    • 토닥이 → 보들보들 핑크 캐릭터

9-3. VRM으로 Export

  1. 좌측 상단 ☰ → Export → Export as VRM
  2. VRM 1.0 선택
  3. 라이선스 화면에서:
    • Avatar Permission: Everyone
    • Commercial Usage: Allow
    • Modification: Allow
    • Redistribution: Allow
  4. 파일명 avatar.vrm 으로 저장

9-4. 프로젝트에 적용

저장한 avatar.vrm 파일을 본인 프로젝트의 public/ 폴더에 복사.

copy "%USERPROFILE%\Desktop\avatar.vrm" "C:\projects\my-bot\public\avatar.vrm"

그리고 푸시:

git add public/avatar.vrm
git commit -m "Add my VRoid avatar"
git push

1~3분 후 Vercel 자동 재배포 → 새 아바타 등장.

10AI 코딩 도구로 페르소나 변경

봇의 말투, 성격, 대화 스타일을 AI 도구로 한 번에 바꿉니다.

10-1. 페르소나 설정 위치

Team Edition 에서는 페르소나(시스템 프롬프트)가 두 곳에 있을 수 있습니다:

  • RAG 청크 자체: 봇 인사말/소개를 첫 청크로 등록 (가장 쉬움)
  • 미들턴 백엔드: 박교수님과 협의해서 팀별 시스템 프롬프트 커스터마이즈 가능

10-2. 청크로 페르소나 정의 (추천)

웹 RAG 관리 페이지에서 첫 청크로:

{"id":"persona-01","question":"안녕","answer":"안녕! 나는 '분개해' 봇이야. 회계 분개 풀어주는 학습 친구야. 어떤 문제 같이 풀어볼래?"}
{"id":"persona-02","question":"너는 누구야","answer":"21학번 F5 팀이 만든 회계 학습 봇 '분개해'예요. 분개 문제 입력하면 차변/대변 풀이와 설명을 드려요."}

10-3. (고급) AI 코딩 도구

코드 자체를 수정하려면:

  • Claude Code (CLI) — 가장 강력
  • ChatGPT — 코드 복붙
  • Google Gemini

Team Edition 의 경우 백엔드(미들턴)에 system prompt 가 있어서, 학생이 직접 수정하긴 어렵습니다. RAG 청크로 페르소나를 표현하는 게 가장 실용적.

🆘 자주 겪는 문제

Q1: 봇 채팅이 작동하지 않습니다

  • Vercel 환경변수 TEAM_ID 가 본인 팀 번호로 설정되어 있는지 확인
  • Vercel Deployments → 최근 배포 로그에서 에러 확인
  • 브라우저 콘솔(F12) 에서 빨간 메시지 확인

Q2: RAG 청크 추가 페이지에서 "403 팀 키가 일치하지 않습니다"

  • 박교수님께 받은 본인 팀 키 정확히 입력했는지 확인
  • 키 형식: team-XX-xxxxxxxxx (XX는 팀 번호)
  • 공백/줄바꿈 없이 복사

Q3: 봇이 RAG 청크를 무시하고 일반 답변만 합니다

  • 청크 추가 후 "총 N개" 메시지 확인 (실제 저장됐는지)
  • 질문이 청크 question 과 의미적으로 비슷한지 (코사인 유사도 임계값)
  • 청크 수가 너무 적으면 (5개 이하) 매칭이 안 잡힐 수 있음 — 30개 이상 권장

Q4: 카카오 공유 버튼이 작동하지 않음

  • Vercel 환경변수 VITE_KAKAO_JS_KEY 확인
  • Kakao Developers 의 Web 플랫폼에 Vercel URL 등록 확인

Q5: VRM 아바타가 안 보입니다 ("아바타가 비어있어요")

  • public/avatar.vrm 파일 존재 + git 에 푸시됐는지 확인
  • Vercel Deployments → Source → public/avatar.vrm 있는지
